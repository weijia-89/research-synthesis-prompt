# References, primary-source registry for `palamedes`

This file is the registry of primary sources cited by the skill in research output. **It starts empty and is populated lazily, on first use, with verified-this-session entries only.**

The skill's own reference docs (`references/*.md`) deliberately use **author-year** citation form (e.g. "Sharma et al. 2023"; "Ioannidis 2005") rather than `[T1-verified]` tags. Reason: those are pedagogical / orientation docs and bootstrapping them with `[T1-verified]` tags before retrieval would replicate the citation-laundering pattern the skill exists to prevent.

When user-facing research output cites a source: retrieve, then add row, then tag.

## Tier definitions (full criteria in `references/source-grading.md`)

- **T1**, peer-reviewed primary, RCT/cohort/meta-analysis, standards body (W3C / IETF / NIST / ISO / OWASP), official spec / statute, original dataset, vendor authoritative spec for that vendor's product.
- **T2**, credible secondary: reputable journalism with named author + linked primaries, established expert blog with track record, vendor blog with named engineer authors, widely-cited textbook in widespread course-adoption.
- **T3**, tertiary, forum, single tweet, anonymous blog. Use only with explicit downgrade note.

## Tag form in research output

```
Claim text [T1-verified, TAG-SHORTHAND]
Claim text [T1-verified, TAG-SHORTHAND, COI:Pfizer]
Claim text [T2-verified, TAG-SHORTHAND, stale:2023-06]
Claim text [inferred:T1+priors], basis named
Claim text [priors-only], forbidden ≥ L3
```

## How to add a row

1. Retrieve URL / DOI this session. Read at the depth implied by `[*-verified, read:<depth>]` tag, see `references/source-grading.md` "Read-depth" section.
2. Determine tier per `references/source-grading.md`.
3. Add row to the correct tier section below. Format: `| TAG | Author Year, Title | URL/DOI | YYYY-MM-DD-verified | read:<depth> |`.
4. Cite using tag.

## How to update / re-verify

- If you re-retrieve a source, update `Verified` column to today's date.
- Source moved / 404 → mark `Verified: dead-link YYYY-MM-DD`, find archive (Wayback, Internet Archive Scholar) and add `Archive: <URL>` column.
- Retraction / correction → mark `RETRACTED YYYY-MM-DD` and never cite for original claim.

## Registry-tag-shorthand convention (read this before opening reference docs)

Reference docs (`references/*.md`) sometimes use bracketed shorthands like `[SHARMA-SYC-2023]` or `[METR-2025]` in body text. **These are not `[T1-verified]` claim tags.** They are *proposed registry shorthands*, the tag the row would carry if/when it gets added below.

When a research session uses one of those names, the operator must:
1. Retrieve the URL/DOI this session.
2. Add a row to the appropriate tier table above with `Verified: <today>` and `Read: <depth>`.
3. Use the *same* shorthand as the tag in the claim: `Claim text [T1-verified, SHARMA-SYC-2023, read:body]`.

Until retrieved, treat any bracketed shorthand in a reference doc as an author-year hint ("Sharma 2023 sycophancy paper, arXiv:2310.13548"), not as evidence.
